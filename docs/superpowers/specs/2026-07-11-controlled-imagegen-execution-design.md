# 受控生图执行合同设计

## 目标

让 CyberPPT 的正文图生成只能消费审批后的逐页 `full.prompt`，并以哈希、逐页运行记录和内容 QA 阻止提示词改写、批量混用及未验收图片进入下游。

## 执行合同

新增项目级 `imagegen-run` 命令。命令只接受项目、页码范围和可选模型参数；从当前 `page_image_pairs.json` 读取指定页的 `full.prompt` 和输出路径。不得接受自由提示词、提示词文件或自定义输出路径。

每次只执行一个内容页。执行前写入运行记录，包含清单路径、页码、`full.prompt` 的 SHA-256、模型、目标尺寸和待写入输出路径。完成后记录输出文件 SHA-256、实际尺寸、时间和状态。输入提示词哈希或清单哈希改变时，旧运行记录失效，图片不得被视为当前工件。

## 质量门

执行器在写入图片后调用项目既有 image-text QA。生成图无法读取、尺寸不匹配、QA 为 `failed` 或 `review_required` 时，运行状态为失败/待复核，不能进入蓝图图片审批、BACKGROUND/TEXT 制备、三图转换或 PPT 装配。

## 兼容性与范围

现有 `final-script-pages` 只生成清单，不改变其产物结构。新的执行器只消费 `pairs[]` 内容页，不处理 `skipped_pages[]` 模板页。当前 `full-image-only` 与显式三图分支共用受控 FULL 图执行记录；三图的 BACKGROUND/TEXT 生成另行遵守对应合同。

## 验收标准

1. 自由提示词、自由输出路径和多页合并请求均无 CLI 入口。
2. 每页运行记录包含输入、模型、尺寸、输出和哈希证据。
3. 改写 `full.prompt` 或清单后，旧图被识别为过期。
4. 非内容页无法被生图命令调度。
5. 未通过文字 QA 的页面不能被图片审批或生产装配接受。
6. 单页成功、提示词漂移、尺寸错误、QA 失败及模板页请求均有自动化测试。
