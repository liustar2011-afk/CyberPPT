# 生图阶段内容页过滤设计

## 目标

在 CyberPPT 的逐页生图清单构建阶段排除由 PowerPoint 品牌模板渲染的导航页，确保只为承载正文业务内容的页面创建图像生成任务。

## 范围

- 排除封面、目录、章节过渡页和封底（统称 `template_only` 页面）。
- 继续由现有模板导出逻辑渲染这些页面的标题、目录、章节标题和结束页。
- 内容页继续使用现有的逐页脚本、内容锁定、风格锁和 full-image 生成合同。

## 设计

建立可复用的页面角色分类接口，输出 `cover`、`agenda`、`section`、`ending` 或 `content`。生图清单构建器仅为 `content` 页面写入 `pairs[]`；导航页写入 `skipped_pages[]`，记录页码、标题、角色和原因 `template_only_page`。

页面范围仍接受用户输入。若范围只包含导航页，命令必须明确报错而不是写入空的图像生成清单。若范围混合页面，生成任务只包含内容页，保留原始请求范围和实际生成页范围，以便后续模板装配完整页面集。

## 验收标准

1. 1、2、3、6、9、17、19 页不出现在本项目的图像生成任务中。
2. 4、5、7、8、10–16、18 页保持原有生成提示词与路径合同。
3. 清单可追溯被跳过的页面及其原因。
4. 全部为导航页的请求会得到可操作的错误信息。
5. 现有模板装配对封面、目录、章节过渡与封底的行为不变。
