# CyberPPT 项目执行流程约束

本文件记录本项目后续执行约束，优先级高于临时对话习惯。

## 用户确认的流程要求

1. 先写脚本或提示词，再执行生成。
2. 形成脚本后必须停下来与用户交互，允许用户修改。
3. 过程中的脚本和最终脚本都要保存下来。
4. 生图脚本必须明文保存，不只保存在对话历史或工具调用记录中。

## 当前项目执行门

### Stage 2 蓝图

- 每页 ImageGen prompt 先写入 `workbench/prompts/stage2-imagegen-prompts.md`。
- 用户确认 prompt 后，才允许调用 ImageGen。
- 生图后记录输出路径、生成批次和必要的质量问题。

### Stage 3 PPTX

- 每页先写入或更新 `workbench/reconstruction/build_slide_XX.js`。
- 生成脚本写完后停下，让用户审阅页面结构、内容、版式策略。
- 用户确认后，才允许运行脚本生成 PPTX、渲染图和 QA 文件。
- 不因已经有蓝图而直接批量生成整套 PPTX。
