## 开放数据冲突、缺失证据和 caveat

- 未发现源材料内部明显数值冲突；主要 caveat 是若干关键数字只有正文口径，未附原始底表或计算过程。
- 投资测算只是参考档位，材料明确最终预算以后续专项论证和财务批复为准。
- 首期场景已多处指向“全国月度与季度供需形势分析 + 年度报告自动化”，但是否压缩到只做这两个场景，仍需用户确认。
- 数据授权边界、现有系统复用程度、已有模型误差表现、人员缺口、外部数据接口可得性均属于待摸底项，不能在 PPT 中写成已具备。
- 场景扩展中涉及高频负荷、场站级风光出力、储能 SOC、虚拟电厂资源、现货出清明细等高敏数据，需以汇总级、脱敏级、样板区域和降级输出作为默认表达。

## 内容脑暴：3 条备选故事线

### 备选 A：领导审定型建设方案

- 核心结论：中电联应将电力供需预测升级为行业级公共预测与预警能力建设工程，并先以月季分析和年度报告自动化形成可验收闭环。
- Issue tree：为什么必须建 -> 建什么能力 -> 先做什么场景 -> 如何推进 -> 需要领导定什么。
- 关键证据：E01-E04、E07-E10、E16-E18、E22。
- Caveat：投资、数据授权和存量系统复用尚需专项摸底。
- 适用受众：中电联领导、部门负责人、跨部门协调会。
- 推荐原因：最贴近源材料“需请领导审定事项”，决策导向最强，适合汇报稿转 PPT。

### 备选 B：数据模型平台能力蓝图型

- 核心结论：预测能力建设的本质是把数据治理、模型栈、平台架构和发布复盘机制工程化。
- Issue tree：数据可信 -> 模型稳健 -> 平台承载 -> 发布可控 -> 运营复盘。
- 关键证据：E07、E08、E12-E15、E21。
- Caveat：容易变成技术方案汇报，弱化领导需要当场审定的组织、路径和资源问题。
- 适用受众：信息化、数据治理、模型算法、平台建设工作组。
- 不推荐作为主线原因：材料虽有丰富技术内容，但最终汇报对象更像领导审定，不宜把技术细节放在首位。

### 备选 C：场景牵引型实施路线

- 核心结论：以场景可交付带动数据、模型、产品和机制迭代，先月季和年度，再预警和专题，最后扩展市场与新型主体。
- Issue tree：首期闭环 -> 二期预警 -> 三期扩展 -> 数据责任与降级运行 -> 验收。
- 关键证据：E09、E10、E13、E16、E18、E21。
- Caveat：如果受众需要先判断“为什么现在做”，该线索的必要性铺垫不足。
- 适用受众：项目推进会、实施启动会、场景工作组。
- 谨慎使用原因：适合作为中段展开，不适合单独承担全篇论证。

## 推荐 SCR

- Situation：中电联长期承担行业统计、运行分析、供需研判和报告发布，具备建设行业公共预测中枢的组织与业务基础。
- Complication：新型电力系统和全国统一电力市场改变了供需平衡机理，预测对象从电量负荷扩展到新能源有效出力、跨区互济、市场交易、需求响应、储能充电负荷和极端天气风险；同时数据口径、授权、版本留痕、模型解释和发布审核成为权威预测的前置条件。
- Resolution：将电力供需预测定位为中电联行业级公共预测与预警能力建设工程，按“治理先行、数据可信；平台承载、批流并举；模型驱动、稳健可释；场景闭环、持续运营”推进，首期锁定全国月度与季度供需形势分析和年度报告自动化，形成一个主闭环、两个支撑能力、三个可验收产品，并请领导审定牵头机制、首期场景、摸底调研、外部协同、建设路径和安全要求。
