# 平台架构、功能模块与版本关系

平台总体架构按照“数据源层—数据接入与可信流通层—数据平台与计算层—特征工程与数据治理层—模型算法与 MLOps 层—业务应用与服务层”的纵向主链建设。可信数据流通体系和安全运维保障体系贯穿全链条。

平台需要统一承载数据接入、口径管理、质量校验、版本留痕、授权管理和安全审计，也要承载特征管理、模型训练、滚动回测、结果生成、模型版本和运行监测。这些能力共同支撑全国总盘、区域供需、关键指标趋势、风险识别、专题分析和会商支撑。

业务服务出口包括月报、季报、年报、专题专报、风险清单、领导驾驶舱、指标查询和预测结果订阅，并配套生成、审校、发布和归档。这样可以把数据、模型和报告流程放在统一平台上运行，减少线下流转带来的版本不一致问题。

模型模块关系需要贯通数据接入、数据平台、特征平台、模型训练、模型注册、批量推理、准实时修正、业务服务和监控治理，重点管理数据版本、特征版本、模型版本和发布版本，确保任何一次发布均可回放、可解释、可追责。

平台具体技术选型、存量系统复用范围和新增建设范围，还需要以后续系统能力摸底和专项论证为准，当前不把技术路线和建设边界固化为最终结论。
