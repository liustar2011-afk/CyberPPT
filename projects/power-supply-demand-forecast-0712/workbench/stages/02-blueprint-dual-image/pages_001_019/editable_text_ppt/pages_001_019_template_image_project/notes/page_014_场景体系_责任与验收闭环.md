# 场景体系、责任与验收闭环

场景建设建议采用统一链条来组织，即数据来源—数据资源—加工过程—服务产品—服务对象—反馈复盘。这样可以把能力建设落实到具体使用场景，避免只建平台、不形成稳定应用。

首期场景仍聚焦月度、季度供需形势分析和年度报告自动化。这几个场景与现有工作衔接度较高，也更适合先验证数据口径、模型流程、人工审校和报告生成闭环。

后续可根据条件拓展重点时段预警及需求响应、储能、市场交易等场景。但这些场景依赖数据授权、模型成熟度和协同机制，不宜作为首期承诺。

每个场景都需要同步明确数据责任、验收指标和降级运行机制。特别是在数据缺失、授权不足或模型结果波动较大时，要有可替代的运行安排，确保工作不断档。
