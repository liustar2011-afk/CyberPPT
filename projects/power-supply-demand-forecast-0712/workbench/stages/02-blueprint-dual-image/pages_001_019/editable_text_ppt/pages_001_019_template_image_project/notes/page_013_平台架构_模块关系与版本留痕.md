# 平台架构、模块关系与版本留痕

平台层面建议形成六层主链，包括数据源、可信接入与流通、数据平台与计算、特征工程与治理、模型与生产管理、业务应用与服务。这样可以把数据进入、加工、建模、发布和服务统一承载起来。

版本管理建议至少覆盖四类：数据、特征、模型、发布。四类版本之间需要能够对应，便于说明某一次报告或结论使用了什么数据、什么特征、什么模型，以及形成了哪个发布版本。

平台不仅是功能承载工具，也要保留可追溯、可回放和可审校记录。对供需预测这类需要持续复盘的工作来说，版本留痕和过程记录与功能建设同样重要。

具体技术选型建议留待后续论证，不在现阶段提前固化。当前更重要的是先明确模块关系、数据流向和审校责任。
