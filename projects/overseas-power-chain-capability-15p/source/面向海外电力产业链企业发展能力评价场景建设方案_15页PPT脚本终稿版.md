## 一、整套页面统一规则

### 1. 页面结构规则

- 标题用判断句或结论句，不再单设总结带。
- 页面中部承担主要信息量。
- 不做机械均分，主链、主框、主关系必须突出。
- 架构页优先采用：左侧输入 / 中部核心中枢 / 右侧输出 / 底部运营逻辑。
- 场景页优先采用：上部主体与数据 / 中部主链流程 / 右侧输出结果。
- 运营页优先采用：左组织机制 / 中服务模式 / 右试点路径。
- 风险页采用矩阵式表达。

### 2. 文字规则

- 框内以业务短句、名词短语、动作短语为主。
- 不写解释性过渡语。
- 不写与标题重复的收句。
- 小标题尽量控制在10—16字以内。
- 正文文字不低于10号，小标题不低于12号。

### 3. 关系表达规则

- 箭头必须清楚表达流向。
- 主链优先横向或纵向一条清晰主线。
- 输入、处理、输出要闭环。
- 涉及更新、修正、反馈的内容，要画回流箭头。

→ 企业海外业务加快拓展
→ 外部合作准入要求提高
→ 现有支撑方式难以适应

### 卡片1｜政策要求持续深化

- 数据基础设施、可信数据空间、数据流通利用政策持续推进
- 企业国际化发展、产业链协同出海、合规经营要求持续提高
- 能力证明、风险识别、结果共享逐步成为制度化要求

### 卡片2｜企业海外业务加快拓展

- 业务类型由工程承包、装备出口延伸至投资开发、规划设计、EPC总包、装备供应、储能系统、运维服务、综合能源服务、供应链协同
- 参与主体由企业单边申报转向与业主、总包方、金融机构、保险机构、采购方、认证机构、法律与E&S/HSE机构多主体交互

### 卡片3｜外部合作准入要求提高

- 投标预审要求系统说明主体资质、项目经验、履约能力、技术装备、关键人员、HSE与合规情况
- 融资保险要求系统说明财务信用、治理结构、履约记录、风险缓释和国别风险应对能力
- 供应链和市场进入要求说明认证资质、供货记录、属地化能力、合作伙伴和准入规则适配情况

### 卡片4｜现有支撑方式难以适应

- 材料分散在国际业务、财务、法务、技术、HSE、供应链等多个部门
- 数据口径不统一，证据与评价事项对应关系不清
- 评价结论缺少统一模型、统一证据链和统一表达方式
- 结果难以在投标、融资、供应链、国别进入、行业监测、国际传播等场景复用

# P15｜围绕边界定位、数据质量、规则统一、场景落地、经营平衡和安全合规建立推进控制机制

## 框内文字终稿

### 上部｜风险控制矩阵

| 风险类型 | 主要表现 | 控制措施 |
|---|---|---|
| 定位偏移 | 容易把行业公共性场景做成单一平台建设或单点业务服务 | 先明边界，围绕能力评价、证据支撑、结果应用、持续运营推进 |
| 数据质量不足 | 字段不完整、口径不一致、证据链不闭合、更新不及时 | 先立目录，明确来源单位、授权依据、质量责任、更新机制 |
| 规则不统一 | 评价口径、结果表达、用途边界不统一 | 先统一评价规则、结果规则、复核规则、应用边界 |
| 场景空转 | 场景铺得过多、链路过长、结果难进入真实业务流程 | 先做少量高频场景样板，先跑通链路再扩面 |
| 经营失衡 | 公益底座投入大、市场化转化弱、持续运营不足 | 分层供给、分层运营，公共服务与市场化服务并行组织 |
| 合规风险 | 授权不足、敏感信息外泄、对外传播边界不清 | 分场景授权、权限控制、脱敏处理、调用审批、全过程留痕 |

### 下部｜推进建议

**建议一**
将本场景纳入电力领域数据基础设施重点应用场景，围绕企业海外发展能力评价形成首批样板

**建议二**
选择10—15家基础较好、需求明确、配合度高的企业开展首批试点，优先围绕企业画像、投标预审、融资保险、供应链准入、国别适配等高频场景推进

**建议三**
由中电联统筹规则和行业协调，数智公司承担数据底座与运营交付，专家委员会提供复核校准，企业和外部机构参与真实应用

**建议四**
同步沉淀评价目录、证据清单、授权文本、结果模板、复核规则、反馈机制和推广路径，形成可复制、可推广的场景建设成果

## 箭头关系

- 风险矩阵不加箭头。
- 下部建议框可不加箭头，或用序号体现递进。
