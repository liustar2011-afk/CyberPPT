# SCR 故事线与页面架构

## 构建全篇论证

使用：

- **Situation：** 受众已经接受的稳定背景。
- **Complication：** 需要决策的变化、张力、缺口、风险或机会。
- **Resolution：** 由证据支持的选择、优先级、能力和行动。

SCR 是论证结构，不是三个装饰标签。每个部分都必须制造继续阅读下一页的理由。用 MBB 标准把关：每一页都要有论点、证据、caveat、含义和承接关系。

## SCR 备选与故事线评估

在确定最终 SCR 前，必须基于内容脑暴结果提出 2-3 个 SCR 备选。每个备选都要说明：

- Situation、Complication、Resolution 各自的一句话表达；
- 最强证据链和关键证据 ID；
- 对管理层决策的指向；
- 最大 caveat 或数据缺口；
- 适合做成高密度页面的原因；
- 为什么不选或为什么推荐。

故事线评估至少比较：

- 论证锋利度：是否有明确判断，而不是材料堆砌；
- 证据强度：关键结论是否有足够数据和来源支撑；
- 页面可视化潜力：是否能形成表格、矩阵、漏斗、趋势、拆解或路线图；
- 信息密度：是否能支撑每页多个信息区，而不是宽松卡片；
- 受众适配：是否匹配用户指定的页数、场景和决策需求。

只有用户确认或明确授权后，才把推荐 SCR 收敛成逐页大纲。

## 创建逐页计划

每一页必须明确：

| 字段 | 要求 |
|---|---|
| 页面角色 | Situation、Complication、Resolution、承接页或收束页 |
| 结论标题 | 可辩护的完整句判断 |
| 详细论证 | 支撑结论的逻辑链，包括哪些前提必须成立 |
| 证据 | 来自证据表的 Source ID |
| Caveat | 缺失数据、仅方向性证据或结论风险 |
| 视觉 | 图表、表格、矩阵、流程、地图或结构化文字 |
| 图表计划 | 指标、比较对象、标签、注释和来源 ID |
| 含义 | 这条证据为什么重要 |
| 承接 | 下一页回答什么问题 |
| 信息密度 | 信息区数量、主图/侧栏比例、表格/注释/图例/微图表密度 |
| 组件清单 | 主图、侧栏、表格、注释框、图例、微图表、页脚和 SO WHAT |

## 结论标题测试

弱标题：

- 市场概览
- 消费者分析
- 渠道策略

强标题：

- 市场增长正在修复，但价值正在向结构性优势赛道转移
- 核心消费者继续扩大，但购买逻辑变得更加分化
- 没有差异化经济性的渠道扩张会稀释利润

强标题必须能被页面证据挑战并守住。

## 证据层级

每页内容按以下顺序构建：

1. 结论标题
2. 主证据
3. 次级证据或细分
4. 解读
5. 业务含义或行动
6. 来源说明

不要只放图表而不说明它证明了什么。

## 页面容量

优先让每页回答一个主问题。高信息密度来自结构化证据，不来自极小字号。

典型页面组件：

- 一个结论标题；
- 一个主图表或分析框架；
- 两到四个支持证据块；
- 一个含义或决策框；
- 简洁来源说明。

高密度页面还要定义：

- 主图和侧栏的大致面积比例；
- 信息区数量和每区承载的证据 ID；
- 表格、注释、图例、微图表、标签和脚注数量；
- 哪些内容必须可编辑，哪些复杂视觉可以在第三阶段保留为图片资产；
- 如果信息不足，明确标记为低密度风险，不要让蓝图阶段自行补内容。

当一页包含多个无关结论时，应拆页。当证据重复且不推进论证时，应合并。

## 推荐大纲输出

制作前先提交：

```text
整体 SCR
SCR 备选与故事线评估
第 1 页 - 结论 / 角色 / 详细论证 / 证据 / caveat / 图表计划 / 视觉 / 含义 / 承接
第 2 页 - 结论 / 角色 / 详细论证 / 证据 / caveat / 图表计划 / 视觉 / 含义 / 承接
...
逐页信息密度和组件清单
开放数据冲突
需要用户决策的问题
```

停止并请求第一次确认。
