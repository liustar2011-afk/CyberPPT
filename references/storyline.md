# 汇报主线与页面架构

## 构建汇报主线与页面组织

先识别材料类型、汇报任务和受众，再确定全篇主线及页面组织。不得固定章节顺序；工作方案、阶段进展、形势研判和专题请示可采用不同组织方式。

需要时可使用 SCR：

- **Situation：** 受众已经接受的稳定背景。
- **Complication：** 需要决策的变化、张力、缺口、风险或机会。
- **Resolution：** 由证据支持的选择、优先级、能力和行动。

SCR 是可选分析结构，不是默认最终呈现标签。每一页都要有与材料相称的页面要点、证据、caveat、含义和承接关系。

## 汇报主线备选与组织方式评估

在确定推荐汇报主线前，必须基于内容脑暴结果提出 2-3 个备选。每个备选都要说明：

- 推荐组织方式及适配理由；
- 需要时使用的 Situation、Complication、Resolution 或其他分析工具；
- 最强证据链和关键证据 ID；
- 对管理层决策的指向；
- 最大 caveat 或数据缺口；
- 适合做成高密度页面的原因；
- 为什么不选或为什么推荐。

故事线评估至少比较：

- 任务适配度：是否服务于汇报目的、受众关注事项和预期决策；
- 证据强度：关键结论是否有足够数据和来源支撑；
- 页面可视化潜力：是否能形成表格、矩阵、漏斗、趋势、拆解或路线图；
- 信息密度：是否能支撑每页多个信息区，而不是宽松卡片；
- 受众适配：是否匹配用户指定的页数、场景和决策需求。

只有用户确认或明确授权后，才把推荐主线收敛成逐页计划。

## 创建逐页计划

每一页必须明确：

| 字段 | 要求 |
|---|---|
| 页面角色 | Situation、Complication、Resolution、承接页或收束页 |
| 页面标题或页面要点 | 与材料类型和汇报任务相称的正式表述，不预设结论式写法 |
| 详细说明 | 支撑页面要点的事实依据、工作安排、责任边界或风险提示 |
| 证据 | 来自证据表的 Source ID |
| Caveat | 缺失数据、仅方向性证据或结论风险 |
| 视觉 | 图表、表格、矩阵、流程、地图或结构化文字 |
| 图表计划 | 指标、比较对象、标签、注释和来源 ID |
| 含义 | 这条证据为什么重要 |
| 承接 | 下一页回答什么问题 |
| 信息密度 | 信息区数量、主图/侧栏比例、表格/注释/图例/微图表密度 |
| 组件清单 | 主图、侧栏、表格、注释框、图例、微图表、页脚和 SO WHAT |

## 页面标题或页面要点测试

弱标题：

- 工作总体考虑
- 当前进展情况
- 下一步重点安排

强标题：

- 电力供需预测工作具备既有基础，需进一步明确数据、模型和协同安排
- 当前重点任务已形成推进条件，仍需统筹解决数据口径和组织协同事项
- 建议按工作任务分阶段推进，并同步建立审核、发布和复盘机制

标题应与材料和任务相称，并能由页面证据、事实或工作安排支撑；不得为追求结论先行而写成口号化判断。

## 证据层级

每页内容按以下顺序构建：

1. 页面标题或页面要点
2. 主证据
3. 次级证据或细分
4. 解读
5. 业务含义或行动
6. 来源说明

不要只放图表而不说明它证明了什么。

## 页面容量

优先让每页回答一个主问题。高信息密度来自结构化证据，不来自极小字号。

典型页面组件：

- 一个结论标题；
- 一个主图表或分析框架；
- 两到四个支持证据块；
- 一个含义或决策框；
- 简洁来源说明。

高密度页面还要定义：

- 主图和侧栏的大致面积比例；
- 信息区数量和每区承载的证据 ID；
- 表格、注释、图例、微图表、标签和脚注数量；
- 哪些内容必须可编辑，哪些复杂视觉可以在第三阶段保留为图片资产；
- 如果信息不足，明确标记为低密度风险，不要让蓝图阶段自行补内容。

当一页包含多个无关结论时，应拆页。当证据重复且不推进论证时，应合并。
